# 🛸 Last Parsec — Game Design Document v1.0

---

## 1. ゲーム概要

| 項目 | 内容 |
|---|---|
| タイトル | Last Parsec |
| ジャンル | 一人称サバイバル / ハードSF |
| 視点 | 3D FPS（3D酔い対策オプションあり） |
| プレイ人数 | 1〜6人（ソロ優先・オンラインマルチ後日実装） |
| トーン | シリアス・リアル寄りSF |

## 2. コアコンセプト

> **「宇宙を漂流する母船で、限られたリソースと乗組員の命を管理しながら、亜空間ドライブを修理して目標惑星を目指す」**

- RAFT × Subnautica × Satisfactory を宇宙で実現
- リソースの連鎖設計による深みのある管理ゲームプレイ
- 設備投資による自動化・快適化の成長曲線
- 頻繁な死亡と乗組員カウントダウンによる緊張感の維持

## 3. ストーリー設定

### 背景
- 時代：惑星間移民が日常化した近未来
- 船名：**USCV Persephone（ペルセポネ）**
- ミッション：未調査惑星「Kepler-9f」への100名の調査移民輸送
- 航行方式：亜高速航行（亜空間ドライブ使用）＋冷凍睡眠

### 事故の経緯
航行中に未知の磁気嵐（宇宙フレア帯）に突入。ドライブ系統・電力系統・生命維持系統が連鎖的に損傷。AIが最小限の乗組員を選出し解凍。プレイヤーはその一人。

### ゲームの目的
亜空間ドライブを修理し、Kepler-9fへの到達を目指す。乗組員100名の命がプレイヤーの判断にかかっている。

---

## 4. 並行目標システム（リソースの安定化）

タスク進行と並行して、プレイヤーは以下のリソース安定化を常に追い続ける。船の成長と引き換えに消費量が増えるため、「安定」は一度達成しても維持し続けなければならない。

### 🫁 酸素の安定供給
| 消費が増えるタイミング | 理由 |
|---|---|
| 設備が増えるごと | 酸素を消費する区画・人員が増加 |
| 船体モジュール拡張ごと | 充填すべき空間が広がる |
| EVA移動距離が延びるごと | 宇宙服の酸素消費が増加 |

### ⚡ 電力の安定供給
| 消費が増えるタイミング | 理由 |
|---|---|
| 設備追加ごと | 各設備が常時電力を消費 |
| 母船拡張ごと | 照明・環境制御の範囲が拡大 |
| 遠心重力リング稼働 | 重力維持に大量の電力が必要 |

### 🛢️ 燃料の安定供給
| 消費が増えるタイミング | 理由 |
|---|---|
| 母船拡張ごと | 船体が重くなりスラスター消費増加 |
| EVA時間延長ごと | 宇宙服スラスターの消費増加 |
| 航行距離が延びるごと | より遠い目標への移動コスト増加 |

### 🍱 食料の安定供給
- 単なる「量の確保」ではなく、食材の質の向上が目標
- 上位の食材ほどHP回復量増加・空腹消費速度低下

## 5. コアループ

```
【安定時の基本サイクル：約15〜30分】

① 問題の把握
   船の損傷・食料/酸素/電力/燃料不足・拡張したいなど
        ↓
② レーダーで目標を探す（制御室）
   必要素材に応じてデブリ群・小惑星帯を探知
        ↓
③ 母船を操作して現地へ移動
   スラスターで母船を手動操縦（燃料消費）
        ↓
④ EVAで素材採集
   母船を目標付近に停泊させ、船外に出て採集
        ↓
⑤ 帰還・クラフト・修理
   素材を格納庫に持ち帰りリソース補充・修理
        ↓
⑥ ①に戻る
```

### 割り込みイベント（ループ中いつでも発生）
| タイミング | イベント |
|---|---|
| 移動中・採集中・クラフト中すべてで | ☀️ 太陽フレア / ☄️ 隕石雨 |
| 常時進行中 | 🔧 亜空間ドライブ修理進捗 |
| 航路上の特定ポイントで | 🌀 スイングバイイベント |

## 6. リソースシステム

### 6-1. 消費リソース

| リソース | 枯渇時の影響 | タイムリミット目安 |
|---|---|---|
| 🫁 酸素 | **即時ゲームオーバー**（全乗組員死亡） | 約2〜3分 |
| 💧 水 | HP減少 → 死亡 | 約10〜15分 |
| 🍱 食料 | HP減少（水より遅い） → 死亡 | 約20〜30分 |
| ⚡ 電力 | 全設備停止（酸素供給も止まる） | 即時停止 |
| 🛢️ 燃料 | 航行・姿勢制御不能 | 即時停止 |

**酸素の特別ルール**
- ゲージ0＝即ゲームオーバー。例外なし
- 冷凍睡眠中の乗組員も解凍された瞬間に呼吸するため全員死亡
- ゲージ残り20%で警告音・HUD赤点滅
- 酸素は全リソースの中で唯一の即時ゲームオーバートリガー

**HPとダメージ設計**
- 水・食料不足のダメージは加算される（複合ペナルティ）
- イベントによる直接ダメージあり
- ダウン状態：HPが0で即死ではなく短時間の猶予あり
  - ソロ：猶予時間内に回復アイテムを使用しなければ死亡
  - マルチ：仲間が蘇生可能（医薬品消費）

---

### 6-2. リソース別・安定供給ロードマップ（各5段階）

#### 🫁 酸素
| 段階 | 設備 | 特徴 |
|---|---|---|
| 1 | 予備酸素ボンベ（初期備蓄） | 消耗品・緊急用 |
| 2 | 電気分解装置 | 水から酸素を生成。電力依存 |
| 3 | 苔培養ユニット | 光合成で補助供給。電力不要 |
| 4 | 植物栽培モジュール | 供給量大。水・遠心重力が必要 |
| 5 | 大型電気分解装置 | 高出力・大量供給。電力消費大 |

- 苔・植物は電力が落ちても酸素を供給するフェイルセーフ
- 理想は電気分解＋植物の併用による二重化

#### ⚡ 電力
| 段階 | 設備 | 特徴 |
|---|---|---|
| 1 | 手回し充電器 | プレイヤーが手動で充電。少量・労力大 |
| 2 | 太陽光パネル | 受動供給。恒星から遠いと効率低下 |
| 3 | 改良型太陽光パネル | 変換効率向上。希少素材が必要 |
| 4 | ジェネレーター | 燃料消費型・高出力 |
| 5 | 核融合発電炉 | 最高出力・安定。核物質処理・冷却剤が必要 |

- 核融合炉は核廃棄物の処理という管理要素が発生

#### 🛢️ 燃料
| 段階 | 設備 | 特徴 |
|---|---|---|
| 1 | 漂流燃料缶の回収 | EVAで手動回収。消耗品 |
| 2 | 水素燃料精製装置 | 電気分解の副産物（水素）を燃料化 |
| 3 | バイオ燃料生成装置 | 農業の廃棄有機物を燃料に変換 |
| 4 | 小惑星氷の現地精製 | 大量の氷から水素を直接精製 |
| 5 | 反物質触媒燃料合成 | 極めて少量の素材で高効率燃料を生成 |

#### 🍱 食料
| 段階 | 設備 | 特徴 |
|---|---|---|
| 1 | 船内非常食 | 初期備蓄。消耗品・補充不可 |
| 2 | 藻類・苔の栽培 | 簡易栽培。遠心重力不要・低栄養 |
| 3 | 農業モジュール（野菜類） | 遠心重力が必要。栄養価中程度 |
| 4 | 農業モジュール（穀物類） | 高カロリー。水消費大・育成時間長い |
| 5 | 調合食（クラフト） | 複数食材を組み合わせ。HP回復+++・空腹消費速度大幅低下 |

**有機物合成器（種の入手方法）**
- 宇宙に種は存在しないため、有機物からの合成が必要
- アミノ酸結晶（彗星・氷小惑星）→ コケ・藻類（酸素特化）
- 有機ポリマー（炭素質小惑星）→ 野菜類（バランス型）
- 複合炭素化合物（希少小惑星）→ 穀物類（食料特化）
- 一度合成した種は自家採種が可能になり以降は素材不要

**農業×酸素のシナジー**
- 植物の光合成により酸素を副産物として放出
- 農業が育つほど酸素供給も安定する
- 水消費は増えるがトレードオフとして食料・酸素が安定

#### 💧 水
| 段階 | 設備 | 特徴 |
|---|---|---|
| 1 | 船内非常用水 | 初期備蓄。消耗品 |
| 2 | EVAで手動採集（氷・水パック） | プレイヤーが直接回収 |
| 3 | 大気水分回収装置 | 船内の湿気・呼気を再利用 |
| 4 | ロボットアームによる自動採集 | 停泊中に氷小惑星から自動採掘 |
| 5 | 航行中水分子コレクター | 航行中に宇宙空間の水分子を自動収集 |

- 水分子コレクターは航行中の方が停泊中より収集効率が高い
- 通過する宙域（星間ガス雲・彗星帯）によって収集量が変化
- 水は飲料・農業・電気分解の三用途で消費される最重要リソース

---

### 6-3. 設備間シナジー（成長の連鎖）

```
水 → 電気分解装置 → 酸素 + 水素
                    水素 → 燃料
遠心重力 → 農業モジュール → 食料 + 酸素（光合成）
                            廃棄有機物 → バイオ燃料
電力 → 全設備を支える根幹
```

### 6-4. 素材リソース（収集・クラフト用）

| 素材 | 入手場所 | 主な用途 |
|---|---|---|
| 金属スクラップ | 漂流物・小惑星 | 修理・拡張全般 |
| 高導電ケーブル | 難易度高めの小惑星 | 電力系統・核融合炉 |
| 有機ポリマー | 炭素質小惑星 | 農業・フィルター |
| アミノ酸結晶 | 彗星・氷小惑星 | 種の合成（コケ・藻類） |
| 複合炭素化合物 | 希少小惑星・深宙域 | 種の合成（穀物類） |
| 冷却剤 | 特殊小惑星 | ドライブ修理・核融合炉冷却 |
| 水（氷） | 小惑星 | 電気分解・農業・燃料精製 |
| 希少鉱石 | 遠方の小惑星（高リスク） | 核融合炉・ドライブ部品 |

## 7. 船体・拡張システム

### 船の成長フェーズ

| フェーズ | 規模 | 特徴 |
|---|---|---|
| 初期（損傷直後） | 中型 | 重力なし・全区画が狭く機能最低限 |
| 中盤（修理・拡張） | 中〜大型 | 一部区画に遠心重力リング追加可能 |
| 終盤（フル拡張） | 大型 | 遠心重力で快適化・自動化設備充実 |

### 遠心重力システム
- 初期は無重力：船内移動にスラスター（燃料）消費が必要
- 遠心重力リングモジュールを追加すると疑似重力が発生
- 重力区画では通常歩行が可能になりスラスター燃料を消費しない
- 遠心リングは電力を消費する（トレードオフ）
- 重力あり区画と無重力区画が混在する船内が探索の面白さを生む

```
【拡張イメージ：船の構成】

初期：
  [コックピット]─[機関室]─[格納庫/EVAロック]
   ↑無重力         ↑無重力   ↑無重力

中盤：
  [コックピット]─[居住リング★]─[機関室]─[格納庫/EVAロック]
   ↑無重力         ↑遠心重力      ↑無重力   ↑無重力

終盤：
  [コックピット]─[居住リング★]─[研究棟★]─[機関室]─[大型格納庫/EVAロック]
                  ↑遠心重力       ↑遠心重力
★＝遠心重力区画
```

### 追加可能なモジュール
| モジュール | 効果 | 必要素材 |
|---|---|---|
| 遠心重力リング | 該当区画を重力化 | 金属スクラップ大量・高導電ケーブル |
| 太陽光パネル | 電力を受動的に補充 | 金属・光学素材 |
| 農業モジュール | 食料・酸素を自動生産 | 有機ポリマー・水・遠心重力 |
| 大型燃料タンク | 燃料最大量を増加 | 金属スクラップ |
| 拡張冷凍ポッド | 乗組員の収容数を増加 | 金属・高導電ケーブル |
| 隕石シールド | 隕石雨ダメージを軽減 | 金属・冷却剤 |

## 8. 母船操縦システム

### ハイブリッド航行モード

| モード | 使用場面 | 操作 | 燃料消費 |
|---|---|---|---|
| 手動操縦 | 近距離（採集・停泊・緊急回避） | WASD＋姿勢制御スラスター | 多め |
| 自動航行 | 長距離（目標天体への移動） | マップで目標指定→自動で加減速 | 少なめ |

### 手動操縦
- コックピットに座ることで操縦モードに入る
- 宇宙空間の慣性物理を再現（止まるためには逆噴射が必要）
- イベント発生時の緊急回避も手動で行う

### 自動航行
- 到着予定時刻・燃料消費量をシミュレーション表示してから承認
- 航行中はプレイヤーが自由に船内を動き回れる（修理・クラフトの時間）
- 自動航行中もイベントは発生する → 緊急時は手動に切り替え

### 停泊・採集フェーズ
- 目標付近に到達したら自動航行終了・手動に切り替わる
- 停泊中も慣性で流されるためスラスターで位置を維持する必要あり
- 採集が終わったら次の目標へ自動航行をセット

### ナビゲーションUI
```
┌──────────────────────────────┐
│  ◎ 現在地                    │
│           ★ 小惑星A          │ ← 素材:金属・氷
│             距離: 近          │   リスク: 低
│                               │
│              ◆ 彗星B         │ ← 素材:アミノ酸
│              距離: 遠         │   リスク: 中
└──────────────────────────────┘
燃料消費予測: ██░░░░  到着予定: 3:20
[ 自動航行 開始 ]  [ キャンセル ]
```

## 9. EVAシステム（船外活動）

- 宇宙服には独立した酸素タンクがあり、船内より消費が速い
- 無重力移動：スラスター噴射式（燃料消費）
- ヘルメットHUDにリソース表示
- テザー（命綱）を使わないと遭難リスクあり
- 漂流物は相対速度で動いており、捕獲にスキルが必要

## 10. イベントシステム

### イベントの種類と対応選択肢

| イベント | 特徴 | 対応選択肢 |
|---|---|---|
| ☀️ 太陽フレア | 予告あり・電力系にダメージ | 修理 / シールド展開 / 回路遮断 |
| ☄️ 隕石雨 | 大規模・予告時間短い | EVA禁止 / 緊急防御 / 区画封鎖 |
| 🪨 岩石群（小惑星帯） | ランダム発生・船体に衝突 | 回避機動 / 隕石シールド / 修理 |
| 💻 システム障害 | 内部発生・静かに進行 | 診断 → 修理のミニゲーム |
| 👻 冷凍ポッド故障 | 乗員死亡リスク | 優先度判断（誰を救うか） |

### イベント設計の哲学
> **「正解はない、トレードオフがある」**
- 回避すると燃料を消費する
- 修理には時間がかかり、その間に別の問題が発生する
- シールドは電力を大量消費する

---

## 11. 乗組員システム

- 初期100名（ワールド設定で変更可能：50〜200名）
- プレイヤーが死亡 → 新たな乗組員が解凍される（コスト：電力+資源）
- 乗組員が0名 → ゲームオーバー
- 乗組員数は船の最大稼働能力にも影響

### プレイヤー人数とゲームバランスの自動スケーリング

| 人数 | 乗組員初期数（推奨） | イベント頻度 |
|---|---|---|
| 1人 | 50名 | 低め |
| 2〜3人 | 75名 | 中程度 |
| 4〜6人 | 100名 | 高め |

### プレイヤーの役職
| 役職 | 能力ボーナス | 主な担当 |
|---|---|---|
| エンジニア | 修理速度+20% | 機関室・ドライブ修理 |
| 医師 | 酸素消費-10%・回復速度+ | 医療室・乗員管理 |
| パイロット | スイングバイ成功率+ | 制御室・航路管理 |
| 科学者 | 素材発見率+ | 素材分析・クラフト効率+ |
| EVA技術者 | EVAスラスター燃料-15% | 船外作業専門 |
| セキュリティ | 隕石防御効率+ | シールド・緊急封鎖対応 |

## 12. マルチプレイ設計方針

### 開発フェーズ
```
Phase 1（現在）：ソロで完全に動くゲームを作る
Phase 2（後期）：Fishnet Networkingでマルチ対応を追加
```

### ソロ設計時に意識すること
- ゲームロジックをサーバー権威型で設計する
- プレイヤーオブジェクトを最初から複数体対応できる構造にする
- リソース・船体状態などのゲーム状態は中央管理（GameManager）に集約する
- ハードコードされた「プレイヤーは1人」という前提をコードに入れない

## 13. ストーリー進行（タスク・マイルストーン方式）

### 設計方針
- SatisfactoryのようにAIナビゲーターがタスクを提示する形式
- 必須タスクと任意タスク（快適化）に分かれる
- 任意タスクをこなすほど生存が楽になるが、必須タスクだけでもクリア可能

### フェーズ一覧

#### 🔴 フェーズ1「緊急対応」
| タスク | 内容 |
|---|---|
| 必須 | 酸素供給システムの応急修理 |
| 必須 | 冷凍ポッドの電力系統修理 |

## 15. UI / UX 方針（Satisfactory風クリーンSF）

### グラフィックスタイル
- 参考：Satisfactory（明るく鮮明・クリーンで視認性重視）
- 船内は照明が当たった明るいSF空間
- カラーパレット：金属のシルバー・白・オレンジのアクセント

### HUD
- リソース情報（酸素・電力・水・燃料・食料）は画面端に常時表示
- アイコン＋数値＋ゲージバーの組み合わせ
- 酸素ゲージは特に大きく・目立つ位置に配置
- ミニマップ or 船内フロアマップを右下に表示
- 3D酔い対策オプション（FOV調整・カメラ揺れオフ・人工水平儀）

### UIデザイン
- フォント：丸みのあるSFサンセリフ体（視認性重視）
- 色調：白ベース＋青/オレンジのアクセントカラー
- インベントリ・クラフトUI：Satisfactory / Subnautica風グリッド型
